Project Documentation: Task Manager
This section contains the architectural design diagrams for the Task Management system. These documents outline the system's requirements, structure, and dynamic behavior.

Design Documents
1. Use Case Diagram
Purpose: Defines the functional boundaries and user interactions.
Key Features: Task creation, priority setting (High/Medium/Low), habit tracking, and status updates.

2. Class Diagram
Purpose: Illustrates the static structure and data models.
Core Classes: Task, Category, User, and PriorityManager.
Logic: Shows how tasks are linked to categories and how attributes are stored in the FastAPI backend.

3. Behavioral Diagram
Purpose: Models the lifecycle of a task and interaction sequences.
Logic: Details the flow from task entry to completion, including the logic for automated sorting by priority.
